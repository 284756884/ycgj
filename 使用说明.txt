Ozon 约仓工具使用说明

更新时间：2026-06-08

一、每次使用前

1. 双击 start-proxy.bat 启动本地代理。
2. start-proxy.bat 会在后台启动代理，正常情况下窗口会自动关闭。
3. 双击打开 约仓工具2.5_店铺SKU名称独立版.html。
4. 如需确认代理是否正常，双击 check-proxy.bat。
5. 不使用时，可双击 stop-proxy.bat 关闭本地代理。

二、店铺接口设置

1. 顶部选择或新增店铺。
2. 点击“接口”按钮。
3. 填写当前店铺的 Client-Id 和 Api-Key。
4. Performance Client ID / Client Secret 可以保存；当前每日销量日历不再使用广告数据接口，可以留空。
5. 点击保存。

说明：
- 每个店铺的接口信息独立保存。
- 信息保存在当前电脑、当前浏览器的本地数据中。
- 换电脑或换浏览器后需要重新填写。
- 导出的 JSON 备份会包含接口密钥，不要发给他人。

三、顶部库存表

1. “仓库”和“汇总”两列已冻结，横向滚动时固定在左侧。
2. “库存”行点击“同步”，会同步当前 SKU 的 Ozon 库存。
3. “月销量”行点击“同步”，会同时执行两件事：
   - 同步各地区近 28 天销量，并写入每个地区月销量输入框；
   - 同步今天起往前 28 天的每日销量日历缓存。
4. 如果之前已经同步过部分每日销量日期，再次同步时只补缺失日期；当天和前一天会重新覆盖，避免当天未结束导致旧数据被永久忽略。
5. 手动修改任意地区月销量输入框时，会清除自动同步的汇总月销量，汇总会重新按地区输入框合计。

四、每日销量日历

1. 点击“汇总”列里的月销量数字，打开每日销量日历。
2. 日历按正常月历展示，每个日期下方显示当天销量。
3. 日历支持上个月、下个月切换。
4. 点击“同步近28天”，只会同步今天起往前 28 天中本地缺失的日期，并覆盖当天和前一天的数据。
5. 未来日期只显示在日历上，不会请求 Ozon 接口。
6. 当前版本不再显示广告占比，也不再调用 Performance 广告数据接口。

五、普通送仓记录

1. 选择店铺和 SKU。
2. 在对应地区填写送仓数量。
3. 可在申请号输入框填写 Ozon 交货申请号。
4. 选择送仓日期。
5. 点击单个地区的“送仓”，或点击“批量送仓”。
6. 记录会进入“当前待送仓”。
7. “当前待送仓”和“已完成送仓历史”之间的边框可以上下拖动，手动调节两个区域高度。

六、交货申请改期

1. 单个申请号改期：
   - 在“当前待送仓”中找到申请号旁边的日历按钮。
   - 点击后会打开日期弹窗，并同时查询该交货申请在 Ozon 后台可更改的日期。
   - 可申请日期会在日历日期下方显示蓝点，并显示可改日期天数。
   - 选择带蓝点的日期后点击确认，工具会提交 Ozon 后台改期接口。
   - 后台改期成功后，页面日期才会同步修改；后台失败则页面日期不变。

2. 统一改期：
   - 在“当前待送仓”的日期行右侧点击“改期”。
   - 弹窗默认选择“更改页面时间，同时更改后台时间”。
   - 如只想改页面显示，可以手动切换为“只改变页面交货时间”。
   - 选择后台改期时，工具会逐个遍历该日期下的交货申请：一个申请改完再改下一个。
   - 单个申请失败不会中断后续申请；全部遍历完成后会显示成功和失败明细。
   - 只有后台改期成功的申请号，页面日期才会同步修改。

七、智能校准箱子

1. 在“当前待送仓”中找到申请号。
2. 点击申请号旁边的齿轮按钮。
3. 输入箱规数量，例如 30 或 60。
4. 确认后，工具会调用 Ozon 接口校准该交货请求的箱子。

校准规则：
- 箱数 = 申请总数量 / 箱规，向上取整。
- 例如 180 个、箱规 30，生成 6 箱，每箱 30。
- 例如 179 个、箱规 60，生成 3 箱，前两箱 60，最后一箱 59。
- 如果 Ozon 里已有箱子，会覆盖写入最终正确箱数和箱内组成。

注意：智能校准会真实修改 Ozon 交货请求里的箱子和组成。

八、当天导出、批量校准、下载 PDF

1. 在“当前待送仓”中找到某个日期行。
2. 点击该日期行右侧的“导出”按钮。
3. 在弹窗里填写：
   - 每箱箱规数量；
   - 文件导出 / PDF 保存目录。
4. 如需下载当天所有标签，保持“生成表格后下载当天全部 PDF 标签”勾选。
5. 点击开始校准并导出。

执行顺序：
1. 逐个校准当天所有带申请号的交货请求。
2. 校准前会检查控制台 SKU 与交货请求内实际 SKU 是否一致。
3. 如果 SKU 不一致，会提示并停止，避免写错交货请求。
4. 查询每个申请的货物 ID。
5. 生成 Excel 表格，并把货物 ID 写入“货位ID”列。
6. 保存 Excel 到填写的目录。
7. 依次生成并下载当天所有交货请求的 PDF 标签。

文件命名：
- Excel：货号-店铺-日期.xlsx
- PDF：申请号.pdf

九、常见问题

1. 提示无法连接本地代理
   请先双击 start-proxy.bat，再双击 check-proxy.bat 检查。

2. 提示缺少 Client-Id 或 Api-Key
   点击顶部“接口”，给当前店铺保存接口信息。

3. 每日销量同步失败，提示 date_to value
   旧版本可能会请求未来日期。当前版本已限制为只请求今天及以前日期。

4. 提示 SKU 不一致
   当前控制台 SKU 与 Ozon 交货请求中的 SKU 不一致。请确认是否选错店铺、选错 SKU 或填错申请号。

5. 后台改期失败或没有蓝点日期
   说明该交货申请在 Ozon 后台当前没有可提交的改期时间段，或接口查询失败。页面日期不会被修改。

6. PDF 长时间没有生成
   Ozon 标签生成有时需要等待。可稍后重新点击导出，或单独重新下载。

7. 保存目录失败
   请填写完整目录路径，例如：
   D:\Ozon导出
   或：
   C:\Users\Administrator\Downloads\OzonLabels

8. Excel 导出失败
   检查电脑是否能访问：
   https://cdn.sheetjs.com/

十、安全提醒

Client-Id、Api-Key、Performance Client ID 和 Client Secret 都属于店铺接口密钥。
不要把带密钥的 JSON 备份、浏览器数据或截图发给他人。
